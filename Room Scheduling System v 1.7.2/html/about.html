Use Case Document

Introduction:

This Use Case document has been developed for Saint Charles Community College’s (SCC) new class scheduling software. This software will replace the dated system that SCC currently uses to schedule classes each semester. SCC staff have requested a more efficient scheduling system in order to reduce the number of man-hours currently required to schedule classes. A.C.K.’s system will be designed to meet this need and improve the overall user experience.

Description

Every semester new classes are added and removed from the curriculum. This program should schedule these classes and allow the user to see where a class is held, which classes were not able to be scheduled and if there is any space left for additional classes.

Actors

Schedule Coordinator, department heads.

Exceptions

If the user does not fill out required information when adding a class or room, the program will notify the user that it was unable to add the class or room and notify them of what information is missing.

Pre-conditions

User should know what classes they want to schedule for the semester, rooms that are available to be scheduled to, special requirements the class needs the room to have, special attributes the room has, and if a teacher of a class is only available between certain times.

Post-conditions

The program will allow the user to see the schedule of individual rooms, the location of individual classes, a list of scheduled classes, a list of unscheduled classes, a list of fully booked rooms and a list of rooms that still have space for new classes.

Flow

The user will enter the classes into the program, filling out information into a form for each class. The user will then enter the rooms into the program, filling out information into a form for each room. The user will then click schedule, which will then schedule the classes into rooms. The user will then be able to search rooms and classes.

Alternative Flows

If the program was unable to schedule a class or classes, then the program will allow the user to see a list of unscheduled classes. If rooms are not considered full by the program, the user will be able to see a list of rooms that can support added classes.
